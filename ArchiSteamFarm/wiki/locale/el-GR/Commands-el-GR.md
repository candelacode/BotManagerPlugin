# Εντολές

Το ASF υποστηρίζει ποικιλία εντολών, οι οποίες μπορούν να χρησιμοποιηθούν για τον έλεγχο της συμπεριφοράς της διαδικασίας και των παρουσιών bot.

Παρακάτω οι εντολές μπορούν να σταλούν στο bot με διάφορους τρόπους:
- Μέσω διαδραστικής κονσόλας ASF
- Μέσω ιδιωτικής /ομαδικής συνομιλίας Steam
- Μέσω της διασύνδεσης **[IPC](https://github.com/JustArchiNET/ArchiSteamFarm/wiki/IPC)**

Λάβετε υπόψη ότι η αλληλεπίδραση ASF απαιτεί από εσάς να είστε επιλέξιμοι για την εντολή σύμφωνα με τα δικαιώματα ASF. Ελέγξτε τις ρυθμίσεις `SteamUserPermissions` και `SteamOwnerID` για περισσότερες λεπτομέρειες.

Commands executed through Steam chat are affected by `CommandPrefix` **[global configuration property](https://github.com/JustArchiNET/ArchiSteamFarm/wiki/Configuration#commandprefix)**, which is `!` by default. Αυτό σημαίνει ότι για την εκτέλεση π.χ. `status` command, you should actually write `! tatus` (ή προσαρμοσμένο `CommandPrefix` της επιλογής σας που έχετε ορίσει). `CommandPrefix` δεν είναι υποχρεωτική όταν χρησιμοποιείτε κονσόλα ή IPC και μπορεί να παραλειφθεί.

---

### Διαδραστική κονσόλα

Το ASF έχει υποστήριξη για διαδραστική κονσόλα, αρκεί να μην εκτελείτε το ASF σε λειτουργία [**`Headless`**](https://github.com/JustArchiNET/ArchiSteamFarm/wiki/Configuration#headless). Απλά πατήστε το κουμπί `c` για να ενεργοποιήσετε τη λειτουργία εντολής, πληκτρολογήστε την εντολή σας και επιβεβαιώστε με την είσοδο.

![Στιγμιότυπο οθόνης](https://i.imgur.com/bH5Gtjq.png)

### Συνομιλία Steam

Μπορείτε να εκτελέσετε την εντολή σε δεδομένο ASF bot επίσης μέσω της συνομιλίας Steam. Προφανώς δεν μπορείτε να μιλήσετε απευθείας με τον εαυτό σας, επομένως θα χρειαστείτε τουλάχιστον έναν άλλο λογαριασμό bot αν θέλετε να εκτελέσετε εντολές που στοχεύουν κυρίως στον εαυτό σας.

![Στιγμιότυπο οθόνης](https://i.imgur.com/IvFRJ5S.png)

Με παρόμοιο τρόπο μπορείτε επίσης να χρησιμοποιήσετε την ομαδική συνομιλία της δεδομένης ομάδας Steam. Λάβετε υπόψη ότι αυτή η επιλογή απαιτεί σωστά ρυθμισμένο `SteamMasterClanID` , σε ποια περίπτωση το bot θα ακούσει για εντολές και στη συνομιλία της ομάδας (και να το ενώσετε αν χρειαστεί). Αυτό μπορεί επίσης να χρησιμοποιηθεί για να "μιλήσετε στον εαυτό σας", καθώς δεν απαιτεί ειδικό λογαριασμό bot σε αντίθεση με την ιδιωτική συνομιλία. Μπορείτε απλά να ορίσετε την ιδιότητα `SteamMasterClanID` στη νέα ομάδα σας, Στη συνέχεια, δώστε στον εαυτό σας πρόσβαση είτε μέσω `SteamOwnerID` ή `SteamUserPermissions` του δικού σας bot. Με αυτόν τον τρόπο το ASF bot (εσείς) θα συμμετέχει στην ομάδα και θα συνομιλεί με την επιλεγμένη ομάδα σας και θα ακούει εντολές από τον δικό σας λογαριασμό. Μπορείτε να συμμετάσχετε στην ίδια ομάδα συνομιλίας για να εκδώσετε εντολές στον εαυτό σας (όπως θα στέλνετε εντολή στο chatroom, και ASF περίπτωση που κάθεται στην ίδια αίθουσα συνομιλίας θα τους παραλάβει, ακόμη και αν αυτό δείχνει μόνο καθώς ο λογαριασμός σας είναι εκεί).

Παρακαλείστε να σημειώσετε ότι η αποστολή εντολής στην ομάδα συνομιλίας λειτουργεί σαν ρελέ. Αν λέτε `redeem X` σε 3 από τα bots που κάθεται μαζί σας στη συνομιλία της ομάδας, θα έχει ως αποτέλεσμα το ίδιο όπως θα έλεγες `εξαργυρώσετε το X` σε κάθε ένα από αυτά ιδιωτικά. Στις περισσότερες περιπτώσεις **αυτό δεν είναι ό, τι θέλετε**και αντ' αυτού θα πρέπει να χρησιμοποιήσετε την εντολή `δοσμένη bot` που αποστέλλεται στο **ένα μόνο bot σε ιδιωτικό παράθυρο**. Το ASF υποστηρίζει ομαδική συνομιλία, όπως σε πολλές περιπτώσεις μπορεί να είναι χρήσιμη πηγή επικοινωνίας με το μοναδικό σας bot, αλλά δεν πρέπει σχεδόν ποτέ να εκτελέσετε οποιαδήποτε εντολή στην ομαδική συνομιλία αν υπάρχουν 2 ή περισσότερα ρομπότ ASF κάθεται εκεί, εκτός αν κατανοείτε πλήρως τη συμπεριφορά του ASF που γράφεται εδώ και στην πραγματικότητα θέλετε να μεταδώσετε την ίδια εντολή σε κάθε μεμονωμένο bot που σας ακούει.

*And even in this case you should use private chat with `[Bots]` syntax instead.*

### IPC

Ο πιο προηγμένος και ευέλικτος τρόπος εκτέλεσης εντολών, ιδανικός για αλληλεπίδραση χρήστη (ASF-ui) καθώς και εργαλεία τρίτων ή σενάρια (ASF API), απαιτεί την εκτέλεση της ASF σε λειτουργία `IPC` , και ένας πελάτης που εκτελεί εντολή μέσω διεπαφής **[IPC](https://github.com/JustArchiNET/ArchiSteamFarm/wiki/IPC)**.

### Σημειώσεις

Όλες οι εντολές είναι ευαίσθητες σε κρούση, αλλά τα επιχειρήματά τους (όπως τα ονόματα bot είναι συνήθως ευαίσθητα στην υπόθεση.

Παράμετροι ακολουθούν τη φιλοσοφία του UNIX, οι αγκύλες `[Optional]` υποδεικνύουν ότι το δοσμένο επιχείρημα είναι προαιρετικό, ενώ οι γωνίες `<Mandatory>` δείχνουν ότι το συγκεκριμένο επιχείρημα είναι υποχρεωτικό. Θα πρέπει να αντικαταστήσετε τα επιχειρήματα που θέλετε να δηλώσετε, όπως `[Bots]` ή `<Nickname>` με πραγματικές τιμές με τις οποίες θέλετε να εκδώσετε την εντολή, παραλείποντας τα στηρίγματα.

`[Bots]` όρισμα, όπως υποδεικνύεται από τις αγκύλες, είναι προαιρετικό σε όλες τις εντολές. Όταν καθορίζεται, η εντολή εκτελείται σε δοσμένα bots. Όταν παραλειφθεί, η εντολή εκτελείται στο τρέχον bot που λαμβάνει την εντολή (Steam chat), ή σύμφωνα με το **[`DefaultBot`](https://github.com/JustArchiNET/ArchiSteamFarm/wiki/Configuration#defaultbot)** ρυθμισμένη ιδιότητα (IPC και διαδραστική κονσόλα).

As an example, `status A` sent to bot `B` is the same as sending `status` to bot `A`, bot `B` in this case acts only as a proxy. Αυτό μπορεί επίσης να χρησιμοποιηθεί για την αποστολή εντολών σε ρομπότ που δεν είναι διαθέσιμα διαφορετικά, για παράδειγμα την έναρξη σταματά bots, ή την εκτέλεση ενεργειών στον κύριο λογαριασμό σας (που χρησιμοποιείτε για την εκτέλεση των εντολών).

**Access** of the command defines **minimum** `EPermission` of `SteamUserPermissions` that is required to use the command, με εξαίρεση το `Owner` το οποίο είναι `SteamOwnerID` ορίζεται στο καθολικό αρχείο ρύθμισης παραμέτρων (και την υψηλότερη διαθέσιμη άδεια).

Πληθυντικά επιχειρήματα, όπως `[Bots]`, `<Keys>` ή `<AppIDs>` σημαίνει ότι η εντολή υποστηρίζει πολλαπλές παραμέτρους συγκεκριμένου τύπου, χωρισμένο με κόμμα. Για παράδειγμα, `κατάσταση [Bots]` μπορεί να χρησιμοποιηθεί ως κατάσταση `MyBot,MyOtherBot,Primary`. Αυτό θα προκαλέσει την εκτέλεση δεδομένης εντολής σε **όλα τα bot στόχων** με τον ίδιο τρόπο που θα στέλνατε την κατάσταση `` σε κάθε bot σε ένα ξεχωριστό παράθυρο συνομιλίας. Please note that there is no space after the comma `,`.

Το ASF χρησιμοποιεί όλους τους χαρακτήρες κενών κενών όσο το δυνατόν οριοθέτες για μια εντολή, όπως το χώρο και τους χαρακτήρες νέας γραμμής. Αυτό σημαίνει ότι δεν χρειάζεται να χρησιμοποιήσετε χώρο για την οριοθέτηση των επιχειρημάτων σας, μπορείτε επίσης να χρησιμοποιήσετε οποιοδήποτε άλλο χαρακτήρα κενών χώρων (όπως καρτέλα ή νέα γραμμή).

Το ASF θα "ενώνει" επιπλέον εκτός εμβέλειας επιχειρήματα στον πληθυντικό τύπο του τελευταίου επιχειρήματος εντός εμβέλειας. Αυτό σημαίνει ότι `redeem bot key1 key2 key3` για `redeem [Bots] <Keys>` θα λειτουργήσει ακριβώς το ίδιο με το `redeem bot key1, ey2,key3`. Μαζί με την αποδοχή της νέας γραμμής ως οριοθέτης εντολής, αυτό σας δίνει τη δυνατότητα να γράψετε το bot `reem` και στη συνέχεια να επικολλήσετε μια λίστα κλειδιών που χωρίζονται από οποιονδήποτε αποδεκτό χαρακτήρα οριοθέτη (όπως newline), ή τυπικό `,` οριοθετημένο ASF. Λάβετε υπόψη ότι αυτό το κόλπο μπορεί να χρησιμοποιηθεί μόνο για παραλλαγή εντολών που χρησιμοποιεί τα περισσότερα ορίσματα (οπότε ο καθορισμός `[Bots]` είναι υποχρεωτικός σε αυτή την περίπτωση, αλλιώς το πρώτο σας κλειδί θα ερμηνευόταν ως το bot προορισμού για να εκτελέσει την εντολή).

Όπως έχετε διαβάσει παραπάνω, ένας χαρακτήρας χώρου χρησιμοποιείται ως οριοθέτης για μια εντολή, επομένως δεν μπορεί να χρησιμοποιηθεί σε επιχειρήματα. Ωστόσο, όπως προαναφέρθηκε, η ASF μπορεί να συμμετάσχει σε επιχειρήματα εκτός εμβέλειας, που σημαίνει ότι είστε πραγματικά σε θέση να χρησιμοποιήσετε ένα χαρακτήρα χώρου στο επιχείρημα που ορίζεται ως ένα τελευταίο για δεδομένη εντολή. Για παράδειγμα, το `nickname bob Great Bob` θα ορίσει σωστά το ψευδώνυμο `bob` bot στο "Great Bob". Με τον ίδιο τρόπο μπορείτε να ελέγξετε τα ονόματα που περιέχουν κενά στην εντολή `κατέχει την εντολή`.

Εντολές που αναμένουν `AppID` ή `SubID` είναι επίσης αρκετά έξυπνες για να εξαγάγετε αυτές τις πληροφορίες από τους συνδέσμους `store.steampowered.com`.

### Όρισμα `[Bots]`

`[Bots]` επιχείρημα είναι μια ειδική παραλλαγή της παράστασης πληθυντικός, καθώς εκτός από την αποδοχή πολλαπλών τιμών προσφέρει επίσης επιπλέον λειτουργικότητα.

Πρώτον και κύριο, το επιχείρημα `[Bots]` υποστηρίζει προκαθορισμένες "ομάδες bot ", οι οποίες ορίζονται ως εξής:

| Όνομα             | Περιγραφή                                                      |
| ----------------- | -------------------------------------------------------------- |
| `@all`, `ASF`     | Όλα τα bots που ορίζονται στην περίπτωση                       |
| `@συλλογή καρτών` | Bots που είναι σήμερα γεωργία                                  |
| `@idle`           | Bots που είναι επί του παρόντος **όχι** γεωργία                |
| `@εκτός σύνδεσης` | Bots that are currently **not** connected to the Steam network |
| `@σε σύνδεση`     | Bots που είναι συνδεδεμένα στο δίκτυο Steam                    |

Για παράδειγμα, η εντολή `status ASF` είναι ίση με την κατάσταση `όλα,σας,bots, listed,here`. Αυτό μπορεί επίσης να χρησιμοποιηθεί για την εύκολη αναγνώριση των bots στα οποία έχετε πρόσβαση, ως λέξη-κλειδί `ASF` , παρά τη στοχοποίηση όλων των bots, θα οδηγήσει σε απάντηση μόνο από εκείνα τα bots που μπορείτε πραγματικά να στείλετε την εντολή. Άλλες ομάδες είναι επίσης διαθέσιμες για να εκτελέσετε ευκολότερα δεδομένη εντολή μόνο σε συγκεκριμένα bots που πληρούν την προϋπόθεση.

`[Bots]` επιχείρημα υποστηρίζει επίσης ειδική σύνταξη "range", η οποία σας επιτρέπει να επιλέξετε μια σειρά από bots πιο εύκολα. Η γενική σύνταξη για `[Bots]` σε αυτή την περίπτωση είναι `[FirstBot]..[LastBot]`. Πρέπει να καθοριστεί τουλάχιστον ένα από τα επιχειρήματα. Όταν χρησιμοποιείτε `<FirstBot>..`, επηρεάζονται όλα τα bots που ξεκινούν από το `FirstBot`. Όταν χρησιμοποιείτε `..<LastBot>`, επηρεάζονται όλα τα bots μέχρι το `LastBot`. Όταν χρησιμοποιείτε `<FirstBot>..<LastBot>`, επηρεάζονται όλα τα bot από `FirstBot` μέχρι `LastBot`. Για παράδειγμα, εάν έχετε bots με το όνομα `A, B, C, D, E, F`, μπορείτε να εκτελέσετε την κατάσταση `B. E`, το οποίο είναι ίσο με την κατάσταση `B,C,D,E` σε αυτή την περίπτωση. Όταν χρησιμοποιείτε αυτή τη σύνταξη, το ASF θα χρησιμοποιήσει αλφαβητική ταξινόμηση για να καθορίσει ποια bots βρίσκονται στο καθορισμένο εύρος σας. Τα επιχειρήματα πρέπει να είναι έγκυρα ονόματα bot που αναγνωρίζονται από το ASF, διαφορετικά η σύνταξη εύρους παραλείπεται εξ ολοκλήρου.

Εκτός από την παραπάνω σύνταξη εύρους, το επιχείρημα `[Bots]` υποστηρίζει επίσης το επιχείρημα **[regex](https://en.wikipedia.org/wiki/Regular_expression)** που ταιριάζει. Μπορείτε να ενεργοποιήσετε το μοτίβο regex χρησιμοποιώντας `r!<Pattern>` ως όνομα bot όπου `r!` είναι ενεργοποιητής ASF για συνδυασμό regex και `<Pattern>` είναι το μοτίβο regex σας. An example of a regex-based bot command would be `status r!^\d{3}` which will send `status` command to bots that have a name made out of 3 digits (e.g. `123` and `981`). Μη διστάσετε να ρίξετε μια ματιά στα έγγραφα **[](https://docs.microsoft.com/dotnet/standard/base-types/regular-expression-language-quick-reference)** για περαιτέρω εξήγηση και περισσότερα παραδείγματα διαθέσιμων μοτίβων regex.

## `ρυθμίσεις απορρήτου`

`<Settings>` όρισμα δέχεται από **έως 7** διαφορετικές επιλογές, διαχωρισμένες ως συνήθως με κανονικό οριοθετημένο κόμμα ASF. Αυτές είναι, κατά σειρά:

| Όργανο | Όνομα             | Εξαρτάται από |
| ------ | ----------------- | ------------- |
| 1      | Προφίλ            |               |
| 2      | OwnedGames        | Προφίλ        |
| 3      | Χρόνος Παιχνιδιού | OwnedGames    |
| 4      | Λίστα Φίλων       | Προφίλ        |
| 5      | Απόθεμα           | Προφίλ        |
| 6      | Δώρα              | Απόθεμα       |
| 7      | Σχόλια            | Προφίλ        |

Για περιγραφή των παραπάνω πεδίων, παρακαλώ επισκεφθείτε το **[Ρυθμίσεις απορρήτου Steam](https://steamcommunity.com/my/edit/settings)**.

Οι έγκυρες τιμές για όλες αυτές είναι:

| Τιμή | Όνομα        |
| ---- | ------------ |
| 1    | `Ιδιωτικό`   |
| 2    | `Μόνο Φίλοι` |
| 3    | `Δημόσιο`    |

Μπορείτε να χρησιμοποιήσετε είτε ένα όνομα χωρίς διάκριση πεζών-κεφαλαίων, είτε μια αριθμητική τιμή. Παράμετροι που παραλείφθηκαν θα είναι προεπιλεγμένο να οριστεί σε `Private`. Είναι σημαντικό να σημειώσετε τις εξαρτήσεις που καθορίζονται παραπάνω, καθώς η συγκεκριμένη καταχώρηση εξαρτάται από τον ορισμό της πιο ανοικτής άδειας που μπορεί να έχει η καταχώρηση.

For example, you **can't** have `Public` playtime setting while having `Private` owned games setting, but you can totally have `Private` owned games setting while having `Public` profile.

### Παράδειγμα

Αν θέλετε να ορίσετε **all** ρυθμίσεις απορρήτου του bot σας με το όνομα `Main` σε `Private`, μπορείτε να χρησιμοποιήσετε οποιοδήποτε από τα παρακάτω:

```text
privacy Main 1
privacy Main Private
```

This is because ASF will automatically assume all other settings to be `Private`, so there is no need to input them. Από την άλλη πλευρά, αν θέλετε να ορίσετε όλες τις ρυθμίσεις απορρήτου σε `Public`, τότε θα πρέπει να χρησιμοποιήσετε οποιοδήποτε από τα παρακάτω:

```text
privacy Main 3,3,3,3,3,3,3,3,3,3
privacy Main Public,Public,Public,Public,Public,Public,Public,Public,Public,Public
```

Με αυτόν τον τρόπο μπορείτε επίσης να ρυθμίσετε ανεξάρτητες επιλογές, ωστόσο σας αρέσουν:

```text
privacy Main Public,FriendsOnly,Private,Public,Public,Public,Private,Public
```

Τα παραπάνω θα ορίσουν προφίλ σε δημόσια, ιδιόκτητα παιχνίδια σε φίλους μόνο, παιχνίδι σε ιδιωτικό, λίστα φίλων σε δημόσια, απογραφή σε δημόσια, απογραφή δώρα σε ιδιωτικά και προφίλ σχόλια στο κοινό. Μπορείτε να επιτύχετε το ίδιο με αριθμητικές τιμές αν θέλετε.

## Άδειες

`addlicense` και `rmlicense` εντολές υποστηρίζουν δύο διαφορετικούς τύπους άδειας, αυτοί είναι:

| Τύπος      | Alias | Παράδειγμα   | Περιγραφή                                                                                 |
| ---------- | ----- | ------------ | ----------------------------------------------------------------------------------------- |
| `εφαρμογή` | `a`   | `app/292030` | Το παιχνίδι καθορίζεται από το μοναδικό `appID`.                                          |
| `υπο`      | `ίνα` | `sub/47807`  | Πακέτο που περιέχει ένα ή περισσότερα παιχνίδια, που καθορίζεται από το μοναδικό `subID`. |

Η διάκριση είναι σημαντική, καθώς το ASF θα χρησιμοποιήσει τη μέθοδο του δικτύου Steam για εφαρμογές και τη μέθοδο του Steam για πακέτα. Αυτά τα δύο δεν είναι συμβατά μεταξύ τους.

Κατά την ενεργοποίηση νέων αδειών, συνήθως θα θέλετε να καθορίσετε την εφαρμογή για δωρεάν Σαββατοκύριακα/μόνιμα παιχνίδια F2P και το πακέτο διαφορετικά.

Για την κατάργηση της υπάρχουσας άδειας, και οι δύο μέθοδοι θα λειτουργήσουν, αν και σας συνιστούμε να καταργήσετε πακέτα για να έχετε ντετερμινιστική και ρητή αίτηση, όπως αλλιώς το Steam ενδέχεται να αφαιρέσει πακέτα που δεν σκοπεύατε να καταργηθείτε, ειδικά αν ο λογαριασμός σας έχει περισσότερα από ένα πακέτα που παρέχουν πρόσβαση στη δεδομένη εφαρμογή. This is because under the hood, license is always identified by the package it grants access to, so if you decide to remove by `appID`, then Steam decides which `subID` it's going to remove.

Συνιστούμε να ορίσετε ρητά τον τύπο κάθε εγγραφής προκειμένου να αποφύγετε αμφιλεγόμενα αποτελέσματα, αλλά για την προς τα πίσω συμβατότητα, αν παραλείψετε να πληκτρολογήσετε εντελώς, το ASF θα υποθέσει ότι ζητάτε το `sub` σε αυτή την περίπτωση. Μπορείτε επίσης να ρωτήσετε μία ή περισσότερες από τις άδειες ταυτόχρονα, χρησιμοποιώντας το πρότυπο ASF `,` οριοθέτησης.

Ολοκληρωμένα παραδείγματα εντολών:

```text
addlicense ASF app/292030
addlicense ASF sub/303386
rmlicense ASF app/292030,sub/303386
```

## Λειτουργίες `εξαργύρωσης ↑`

`εξαργυρώστε ↑` εντολή σας επιτρέπει να ρυθμίσετε τις λειτουργίες που θα χρησιμοποιηθούν για ένα σενάριο εξόφλησης. This works as temporary override of `RedeemingPreferences` **[bot config property](https://github.com/JustArchiNET/ArchiSteamFarm/wiki/Configuration#bot-config)**.

`<Modes>` όρισμα δέχεται πολλαπλές τιμές λειτουργίας, διαχωρισμένες ως συνήθως με κόμμα. Οι διαθέσιμες τιμές λειτουργίας καθορίζονται κατωτέρω:

| Τιμή | Όνομα                 | Περιγραφή                                                                                     |
| ---- | --------------------- | --------------------------------------------------------------------------------------------- |
| FAWK | ForceAssumeWalletKey  | Δυνάμεις `AssumeWalletKeyOnBadActivationCode` που εξαργυρώνουν την προτίμηση για ενεργοποίηση |
| FD   | ΔύναμηΔιανομή         | Δυνάμεις `Διανομή` εξαργύρωση προτίμησης για ενεργοποίηση                                     |
| FF   | Εξαναγκασμός          | Δυνάμεις `Διαβίβαση` εξαργύρωση προτίμησης για ενεργοποίηση                                   |
| FKMG | ForceKeepMissingGames | Δυνάμεις `Διατήρηση των Αγώνων` εξαργύρωση της προτίμησης για ενεργοποίηση                    |
| SAWK | SkipAssumeWalletKey   | Δυνάμεις `AssumeWalletKeyOnBadActivationCode` εξαργύρωση προτίμησης για απενεργοποίηση        |
| SD   | Παράλειψη Διανομής    | Δυνάμεις `Διανομή` εξαργύρωση προτίμησης για απενεργοποίηση                                   |
| SF   | Παράλειψη             | Δυνάμεις `Προώθηση` εξαργύρωση προτίμησης για απενεργοποίηση                                  |
| SI   | SkipInitial           | Παράλειψη εξαργύρωσης πλήκτρων στο αρχικό bot                                                 |
| SKMG | SkipKeepMissingGames  | Forces `KeepMissingGames` redeeming preference to be disabled                                 |
| V    | Validate              | Επικυρώνει τα κλειδιά για σωστή μορφή και παραλείπει αυτόματα τα άκυρα                        |

Για παράδειγμα, θα θέλαμε να εξαργυρώσουμε 3 κλειδιά σε οποιοδήποτε από τα bots μας που δεν έχουν ακόμα παιχνίδια, αλλά όχι το `primary` bot μας. Για την επίτευξη αυτού μπορούμε να χρησιμοποιήσουμε:

`redeem↑ πρωτογενές FF,SI key1,key2,key3`

Είναι σημαντικό να σημειωθεί ότι η προηγμένη εξαργύρωση παρακάμπτει μόνο τις `RedeemingPreferences` που εσείς **καθορίστε στην εντολή**. For example, if you've enabled `Distributing` in your `RedeemingPreferences` then there will be no difference whether you use `FD` mode or not, because distributing will be already active regardless, due to `RedeemingPreferences` that you use. Αυτός είναι ο λόγος για τον οποίο κάθε βίαια ενεργοποιημένη παράκαμψη έχει επίσης ένα με βίαια αναπηρία, μπορείτε να αποφασίσετε τον εαυτό σας αν προτιμάτε να παρακάμψετε τα απενεργοποιημένα με ενεργοποιημένη ή το αντίστροφο.

## Εντολή `εισόδου`

Η εντολή `εισόδου` μπορεί να χρησιμοποιηθεί μόνο στη λειτουργία `Headless` , για την εισαγωγή δεδομένων μέσω **[IPC](https://github.com/JustArchiNET/ArchiSteamFarm/wiki/IPC)** ή Steam όταν εκτελείται το ASF χωρίς υποστήριξη για αλληλεπίδραση χρήστη.

Γενική σύνταξη είναι `εισόδου [Bots] <Type> <Value>`.

`<Type>` είναι διάκριση πεζών-κεφαλαίων και ορίζει τον τύπο εισόδου που αναγνωρίζεται από το ASF. Επί του παρόντος το ASF αναγνωρίζει τους ακόλουθους τύπους:

| Τύπος                   | Περιγραφή                                                                               |
| ----------------------- | --------------------------------------------------------------------------------------- |
| Είσοδος                 | `SteamLogin` bot config property, αν λείπει από το config.                              |
| Κωδικός                 | `SteamPassword` bot config property, αν λείπει από το config.                           |
| SteamGuard              | Ο κωδικός ταυτοποίησης στάλθηκε στο e-mail σας αν δεν χρησιμοποιείτε το 2FA.            |
| SteamParentalCode       | `SteamParentalCode` bot config property, αν λείπει από το config.                       |
| TwoFactorAuthentication | Διακριτικό 2FA που παράγεται από το κινητό σας, αν χρησιμοποιείτε 2FA αλλά όχι ASF 2FA. |

`<Value>` είναι τιμή που έχει οριστεί για δεδομένο τύπο. Προς το παρόν όλες οι τιμές είναι συμβολοσειρές.

### Παράδειγμα

Ας πούμε ότι έχουμε ένα bot που προστατεύεται από το SteamGuard σε λειτουργία μη-2FA. Θέλουμε να ξεκινήσουμε αυτό το bot με `Headless` που να είναι `true`.

Για να το κάνουμε αυτό, πρέπει να εκτελέσουμε τις ακόλουθες εντολές:

`start MySteamGuardBot` -> Bot will attempt to log in, fail due to AuthCode needed, then stop due to running in `Headless` mode. Το χρειαζόμαστε αυτό για να κάνουμε το δίκτυο Steam να μας στείλει τον κωδικό ελέγχου στο e-mail μας - αν δεν υπήρχε ανάγκη για αυτό, θα παραλείψουμε αυτό το βήμα εντελώς.

`είσοδο MySteamGuardBot SteamGuard ABCDE` -> We set `SteamGuard` εισαγωγή `MySteamGuardBot` bot σε `ABCDE`. Φυσικά, `ABCDE` σε αυτή την περίπτωση είναι ο κώδικας που πήραμε στο e-mail μας.

`ξεκινήστε το MySteamGuardBot` -> Ξεκινάμε ξανά το (stopped) bot μας, αυτή τη φορά χρησιμοποιεί αυτόματα τον κωδικό ταυτοποίησης που έχουμε ορίσει στην προηγούμενη εντολή, σωστά συνδεόμενος στη συνέχεια καθαρίζοντάς τον.

Με τον ίδιο τρόπο μπορούμε να έχουμε πρόσβαση σε ρομπότ που προστατεύονται από 2FA (αν δεν χρησιμοποιούν το ASF 2FA), καθώς και να ορίσουμε άλλες απαιτούμενες ιδιότητες κατά τη διάρκεια του χρόνου εκτέλεσης.
